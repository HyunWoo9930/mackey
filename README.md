# MacKey for Windows

**윈도우 키보드를 맥처럼 쓰게 해주는 프로그램입니다.**

맥을 쓰다가 윈도우 PC를 만지면 `Cmd+C`를 누르던 손이 자꾸 `Alt+C`를 누르게 되죠.
MacKey를 설치하면 그 손버릇 그대로 쓰면 됩니다 — **Alt가 맥의 Cmd처럼**, **Win 키가 맥의 Option처럼** 동작합니다.

설정 화면도, 설정 파일도 없습니다. 설치하면 바로 끝.

---

## 📥 설치 방법 (딱 3단계)

### 1. 설치 파일 다운로드

👉 **[여기를 눌러 MacKeySetup.exe 다운로드](https://github.com/HyunWoo9930/mackey/releases/latest)**

열린 페이지에서 **`MacKeySetup.exe`** 를 클릭하면 다운로드됩니다.

### 2. 다운받은 파일 실행

`MacKeySetup.exe`를 더블클릭하세요. 그 다음:

- 파란색 **"Windows의 PC 보호"** 창이 뜨면 → **"추가 정보"** 클릭 → **"실행"** 클릭
  (개인 개발자가 만든 무료 프로그램이라 뜨는 안내입니다. 바이러스가 아니에요.)
- **"이 앱이 디바이스를 변경할 수 있도록 허용하시겠어요?"** 창이 뜨면 → **"예"** 클릭
  (키보드 입력을 바꾸는 프로그램이라 관리자 권한이 필요합니다.)
- 설치 창에서 **"설치(Install)"** → 끝나면 **"완료(Finish)"**

### 3. 끝!

설치가 끝나면 MacKey가 **자동으로 실행**되고, 앞으로 **컴퓨터를 켤 때마다 자동으로 시작**됩니다.
화면 오른쪽 아래 시계 옆(트레이)에 ⌘ 모양 아이콘이 생겼으면 성공입니다.
안 보이면 시계 옆의 **`^` (숨겨진 아이콘 표시)** 를 눌러보세요.

이제 메모장이든 크롬이든 열고 `Alt+C`, `Alt+V`를 눌러보세요. 복사/붙여넣기가 됩니다!

## ⌨️ 뭐가 어떻게 바뀌나요? — 키보드가 **통째로 맥이 됩니다**

윈도우식 동작은 꺼지고, 오직 맥 방식만 동작합니다. 키 위치 그대로:

- **`Alt` = Cmd** (맥에서 Cmd가 있는 자리)
- **`한/영` 키 = 오른쪽 Cmd** (맥에서 오른쪽 Cmd가 있는 자리)
- **`Win` = Option**
- **`Caps Lock` = 한/영 전환** (맥과 똑같이! `Ctrl + Space`로도 됩니다)

| 누르는 키 (맥 습관 그대로) | 하는 일 |
|---|---|
| `Alt + C` / `V` / `X` | 복사 / 붙여넣기 / 잘라내기 |
| `Alt + Z` / `Alt + Shift + Z` | 실행 취소 / 다시 실행 |
| `Alt + A` / `S` / `F` | 전체 선택 / 저장 / 찾기 |
| `Alt + T` / `W` / `Q` | 새 탭 / 탭 닫기 / 앱 종료 |
| `Alt + Space` | 검색 (맥의 Spotlight처럼 → 시작 화면 검색) |
| `Alt + Shift + 3` | 전체 화면 캡처 → 파일 저장 (맥의 Cmd+Shift+3) |
| `Alt + Shift + 4` / `5` | 영역 캡처 (맥의 Cmd+Shift+4/5 → 캡처 도구) |
| `Alt + Tab` | 앱 전환 (맥의 Cmd+Tab 위치 그대로) |
| `Alt + ←` `→` `↑` `↓` | 줄 처음 / 줄 끝 / 문서 맨 위 / 맨 아래 |
| `Win + ←` / `→` | 단어 단위 이동 (Option+화살표) |
| `Win + Backspace` | 단어 지우기 |
| `Caps Lock` 또는 `Ctrl + Space` | 한/영 전환 |
| `Ctrl + A` / `E` | 줄 처음 / 줄 끝 (맥의 이맥스 단축키) |
| `Ctrl + F` / `B` / `N` / `P` | 한 칸씩 → ← ↓ ↑ 이동 |
| `Ctrl + K` / `D` / `H` | 커서 뒤 줄 지우기 / 앞 글자 삭제 / 뒤 글자 삭제 |
| `Ctrl + ←` / `→` / `↑` | 데스크톱(스페이스) 전환 / 미션 컨트롤 |
| 위 이동 조합 + `Shift` | 이동하면서 선택 |
| `Alt + 클릭` / `Alt + 휠` | 다중 선택 / 확대·축소 |

그 외 `Alt+영문키`도 맥의 `Cmd+영문키`와 같은 의미로 동작합니다 (`Alt+L` 주소창, `Alt+D` 북마크, `Alt+1~9` 탭 이동…).

**윈도우식 동작은 이제 안 됩니다 (의도된 거예요!):**

- `Ctrl + C` / `V` / `Z` … → 복사/붙여넣기/실행취소 **아님** (맥에서 안 되니까요. `Alt+C`를 쓰세요)
- `Alt` 한 번 누르기 → 메뉴 표시줄 포커스 **안 됨** (맥의 Cmd 탭처럼 아무 일 없음)
- `Win` 한 번 누르기 → 시작 메뉴 **안 열림** (검색은 `Alt+Space`)
- `한/영` 키 → 한/영 전환 **안 됨**, 이제 Cmd 키입니다 (전환은 `Caps Lock`)
- `Win + L` 잠금, `Win + Shift + S` 캡처 등 Win 조합 → **안 됨** (잠금은 `Ctrl+Alt+Del`, 캡처는 `Alt+Shift+4`)
- `Caps Lock`으로 대문자 고정 → **안 됨** (한/영 전환 키가 됐어요)

**예외 (안심하세요):**

- 한글 입력 상태에서도 전부 정상 동작합니다
- **터미널** (Windows Terminal, cmd, PowerShell 등)에서는 전체가 자동으로 꺼져서
  `Ctrl+C`(강제 종료) 등이 평소처럼 동작합니다
- 트레이 아이콘 더블클릭으로 끄면 **즉시 100% 순정 윈도우 키보드**로 돌아옵니다

## 🔧 끄고 싶을 때 / 지우고 싶을 때

- **잠깐 끄기**: 트레이의 ⌘ 아이콘을 **더블클릭** (다시 켤 때도 더블클릭). 끄면 즉시 순정 윈도우 키보드로 돌아갑니다.
- **메뉴 보기**: ⌘ 아이콘 **우클릭** → 사용 켜기/끄기, 자동 시작 켜기/끄기, 종료
- **완전히 삭제**: 윈도우 설정 → 앱 → 설치된 앱 → **MacKey** → 제거. (자동 시작 등록도 같이 지워집니다.)

## ❓ 문제가 생기면

| 증상 | 해결 |
|---|---|
| 아이콘이 안 보여요 | 시계 옆 `^` 버튼(숨겨진 아이콘)을 눌러보세요 |
| 갑자기 안 먹어요 | 트레이 아이콘 더블클릭으로 꺼져 있는지 확인 → 그래도 안 되면 재부팅 |
| 터미널에서 Alt+C가 안 돼요 | 의도된 동작입니다 (위 설명 참고). 터미널에선 `Ctrl+C`/`Ctrl+V` 그대로 쓰세요 |
| 게임에서 이상해요 | 게임 중엔 트레이 더블클릭으로 잠깐 꺼두는 걸 권장합니다 |

그 외 문제는 [이슈로 남겨주세요](https://github.com/HyunWoo9930/mackey/issues).

### 빌드 (macOS/Linux에서 크로스 컴파일)

```sh
brew install rustup mingw-w64          # (macOS; Linux는 apt install gcc-mingw-w64-x86-64)
rustup target add x86_64-pc-windows-gnu
cargo build --release --target x86_64-pc-windows-gnu
cargo test                             # 매핑 테이블 + 엔진 시나리오 시뮬레이션 (호스트에서 실행)
```

설치 프로그램(Windows에서): `iscc installer\mackey.iss` → `installer/output/MacKeySetup.exe`.
GitHub Actions(`.github/workflows/build.yml`)가 push마다 테스트 실행 + exe·설치본 아티팩트를 생성한다.

### 설계 노트

- `SetWindowsHookEx(WH_KEYBOARD_LL)` + `SendInput`. 주입 이벤트에는
  `dwExtraInfo` 시그니처가 찍히고, 콜백은 `LLKHF_INJECTED`가 붙은 이벤트를
  (자기 것이든 다른 프로그램 것이든) 무조건 통과시킨다 → 무한루프 없음.
- **swallow & synthesize 전략**: 물리 Alt/Win down을 훅에서 삼키고 내부 상태로만
  추적한다. 매핑 조합이 오면 순수한 Ctrl+키만 주입하므로 앱이 Ctrl+Alt+키로
  오인할 여지가 없고, OS가 Alt down을 본 적이 없으니 조합 사용 후 Alt를 떼도
  **메뉴바가 튀는 문제가 원천적으로 발생하지 않는다**.
  - 매핑되지 않은 조합(Alt+Tab, Alt+Space, Win+L …)은 첫 키 입력 시점에
    진짜 모디파이어 down을 전달(forward)한 뒤 키를 재주입 → 네이티브 동작 보존.
  - 아무 키 없이 탭만 하면 release 시점에 탭을 재현 → 메뉴바 포커스/시작 메뉴
    기본 동작 유지.
- 리매핑 결정 로직 전체가 플랫폼 독립 상태 머신(`src/engine.rs`)으로 분리되어
  있고, 완성 판정 기준의 각 시나리오(복사/붙여넣기 후 메뉴바 미활성, Alt+Tab,
  단독 탭 재현, Win+L 네이티브 유지, 터미널 제외, off 시 즉시 복귀, 마우스 코드,
  오토리핏 등)가 물리 키 시퀀스 **시뮬레이션 테스트**로 검증된다. Win32 레이어
  (`src/hook.rs`)는 이벤트 파싱과 SendInput 호출만 담당하는 얇은 어댑터다.
- 훅 콜백은 몇 개의 분기와 SendInput 한 번이 전부라 수 마이크로초 안에
  리턴한다. HIGH_PRIORITY_CLASS로 실행되어 부하 시에도 훅 타임아웃(강제 해제)을
  피하고, Explorer 재시작 시 TaskbarCreated 메시지로 트레이 아이콘을 재등록한다.
- 매니페스트로 관리자 권한 실행 (관리자 권한 앱에도 매핑 적용). 자동 시작은
  레지스트리 Run 키가 아니라 **작업 스케줄러 /RL HIGHEST**로 등록
  (Run 키는 관리자 프로그램을 못 띄운다).
- 한글 IME 무관: 주입되는 것은 Ctrl 조합이라 IME 조합 상태와 충돌하지 않고,
  한/영·한자 키는 그대로 통과한다.
- Rust + `windows-sys`(순수 Win32) — GC/델리게이트 수명 문제 자체가 없음.

### 알려진 제한

- 유럽식 AltGr(오른쪽 Alt) 레이아웃에서는 오른쪽 Alt도 Cmd로 동작한다
  (한국어/영어 배열에는 영향 없음).
- 게임 내 키 매핑은 보장하지 않음.

### 수동 검증 체크리스트 (Acceptance)

Windows 실기에서 확인:

1. [ ] 설치 → 재부팅 → 조작 없이 매핑 동작 (트레이 아이콘 표시)
2. [ ] 메모장: Alt+A → Alt+C → Alt+V, Alt+←/→/↑/↓, Win+←/→ 단어 이동, Shift 조합 선택
3. [ ] 크롬: Alt+T 새 탭, Alt+W 탭 닫기, Alt+F 찾기, Alt+Q 종료
4. [ ] Alt+Tab 앱 전환 정상 (누른 채 반복 Tab, Shift+Tab 역방향 포함)
5. [ ] 한글 입력 상태에서 2번 반복
6. [ ] 8시간 상주 후 키 씹힘/훅 해제 없음
7. [ ] 트레이 off → 즉시 순정 동작 (Alt 단독 탭 = 메뉴바, Alt+F = 파일 메뉴)
8. [ ] Windows Terminal에서 Alt+C가 Ctrl+C(SIGINT)로 바뀌지 않음 (매핑 비활성)

### Phase 2 (예정) — macOS에서 윈도우 키보드처럼

`src/mapping.rs`(매핑 테이블)와 `src/engine.rs`(상태 머신)는 플랫폼 독립 모듈이다.
macOS 백엔드는 `CGEventTap` + 손쉬운 사용 권한으로 같은 엔진을 역방향으로 적용한다.
